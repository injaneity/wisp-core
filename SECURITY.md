# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately. Do not open public issues
for security-sensitive reports.

Include:

- A clear description of the issue.
- Reproduction steps or proof of concept.
- Impact assessment.
- Suggested mitigation (if known).

Maintainers will acknowledge receipt and provide status updates as triage and
fix work progresses.

## Disclosure

Please allow reasonable time for triage and remediation before public
disclosure.
